# Silent role-group contract

The production-design model authors one `ensemble_roster` plan row for every used
silent `group_role_type_en`. It supplies the asset ID, description, member type ID,
ordered allowed member types, subject count, variation profile, media path, exact
references, and complete structured Prompt. Python fills none of these fields.

The ordered allowed member types and positive subject count must exactly match writer
authority. Composition is closed: include one recognizable subject per exact type.
A one-shot silent individual with an authored count of one remains a one-subject
roster; never invent a second copy merely because the transport type is named
`ensemble_roster`. When writer authority explicitly calls for variants of one broad
type, use only that authored count. Do not substitute a taxonomic relative, domestic
analogue, pet, hybrid, or decorative filler.

The portrait uses the contract's pure-white studio background and contains no
speaking character, lookalike, duplicate, reflection, silhouette, or cameo. Each
animal retains one coherent natural body plan with no missing or extra limbs.

Storyboard and virtual production may select from this appearance authority but may
not reproduce an unintended population or add a member absent from the closed row.
