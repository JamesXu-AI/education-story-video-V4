# Direct Prompt and Asset Resolution

Seedance Master may describe readable subjects and reference purposes in whatever
Prompt form it finds effective. The private Segment plan—not parsed Prompt
wording—maps provider media roles to `assets/assets.json` namespaces.

Before provider submission, validate that every manually authored provider token
resolves to an existing current catalog asset of the declared media kind and that
the Location, temporal evidence, and population responsibilities agree with the
private binding plan. Reject missing, mistyped, stale, or contradictory inputs
directly in command output.

For a `location_master`, compare `fixed_set_elements_en`,
`embedded_npc_asset_ids`, and `independent_performer_asset_ids` with the private
binding and Storyboard authorities. Reject a duplicated embedded NPC or an
independently directed role left inside the Location authority. Do not search
Prompt prose for a required Location sentence or vocabulary.

Character anatomy and body topology are private catalog authority. Validate them
against the selected image asset; never serialize their JSON, field names, IDs, or
validation contract into the model-facing Prompt. Seedance Master decides what
observable appearance or movement language helps the current generation.

Do not create compatibility packets, review drafts, receipts, rework requests, or
other approval artifacts. A changed Prompt, private binding, media URI, or
predecessor attempt simply requires rematerialization from current sources. Code
may validate and reject private transport facts; it may not parse or rewrite the
Prompt, add creative requirements, or silently substitute another asset.
