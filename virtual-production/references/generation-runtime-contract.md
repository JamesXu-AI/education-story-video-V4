# Generation runtime

Provider attempts and clips live under `.pending/virtual-production/`. Every
execution plan freezes its exact natural-language Prompt hash, private Segment-plan hash, Storyboard hash,
Seedance parameters, token ordering, asset URLs, dependency wave, and predecessor
attempt identity. A changed Prompt, private binding, or catalog semantic row
invalidates resume/reuse without imposing any Prompt-prose schema.

Prepare runtime media only after preflight. A complete predecessor preserves its
audio for true video extension. A settled matched cut uses the approved provider
last frame as an image reference. Both requests also carry the model-authored
Location-master world binding; runtime code may never infer or substitute it. Cached runtime
media is rejected when its execution-plan hash or predecessor attempt changes.

Retries repeat the identical request at most three times. A successful output has
readable video, Seedance-native audio, and a provider-returned last frame.
