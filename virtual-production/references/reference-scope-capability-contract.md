# Reference and capability contract

Seed Master owns the atomic manifest and Shot-local reference scope. Virtual
production only resolves each unique literal token. The namespace before the first
`.` in an atomic element is an exact asset-catalog ID, `roster--member-type`, or the
reserved `continuity` namespace.

Every token resolves once, token numbers are contiguous per media kind, and media
counts remain within the refreshed provider profile. The runtime media list order
is the sole mapping from literal Prompt token to provider value.

Resolution never implies semantic compatibility. Inspect the actual resolved
media against all active Shot responsibilities and source requirements. Reject a
reference whose visible or audible facts conflict with the required identity,
appearance/injury, costume, prop, occupancy, environment, group, voice, sound, or
action state even when its namespace and URL are structurally valid.
