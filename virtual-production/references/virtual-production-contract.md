# Virtual-production contract

Virtual production consumes Seed Master's native Route A `storyboard.md` and
`storyboard-compile-manifest.json`, then invokes Seed Master Route B. It persists
the returned `segment-NNN.md` Scripts unchanged and never reads or recreates
`storyboard.data.json`.

Creative authority stays in the Storyboard and Route B Scripts. Provider transport
authority stays in `seedance-execution-plans/segment-NNN.json`: exact Script/hash,
shooting-plan fields, Seedance parameters, and one concrete value for every literal
`@ImageN/@VideoN/@AudioN`. Static values come from the current asset catalog;
dependent values come from the exact frozen predecessor attempt.

Before those values become a plan, virtual production freezes the final Prompt,
provider-token responsibilities, selected `assets.json` semantic rows, and owned
Storyboard requirements, then requires a current semantic PASS receipt. It does
not download media or compare provider/local file bytes. Any identity,
appearance/injury, costume, prop, occupancy, group, environment, light/color,
voice/sound, or action-phase declaration conflict blocks the plan and emits an
asset rework request owned by direct production design.

Parallel Scripts may execute together. Dependent Scripts must be reviewed and
recompiled by Seed Master as `observed_adapted` before materialization or provider
submission.
